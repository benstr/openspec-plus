Pillars (openspec-plus): before planning or authoring any artifact, read
openspec/pillars/DEFINITIONS.md (controlled vocabulary), then
openspec/pillars/README.md (authority model and reading order).
Pillars govern specs — specs enrich pillars, never contradict them.
Prefer canonical terms when authoring; reading stays permissive via aliases.
Never coin an undefined word — a needed concept gets its DEFINITIONS.md
entry in the same change (/opsx:definition). Pillar meaning changes are
loud and owner-gated: supersede with a pointer, never silently rewrite.
